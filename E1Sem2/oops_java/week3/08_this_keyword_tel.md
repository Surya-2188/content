
#  `this` కీవర్డ్ - పూర్తి లెక్చర్ (రాజు - రవి ఉదాహరణ)


##  1️⃣ అసలు `this` ఎందుకు కావాలి? (ఒక చిన్న బ్యాంక్ కథ)


మనం ఒక చిన్న బ్యాంక్ సాఫ్ట్‌వేర్ తయారు చేస్తున్నాం.
ప్రతి `Account` ఆబ్జెక్ట్ ఒక మనిషిని సూచిస్తుంది. వాళ్ళ దగ్గర **పేరు (Name)** మరియు **డబ్బు (Balance)** ఉంటాయి.


ఇప్పుడు మనం ఇద్దరు నిజమైన వ్యక్తుల అకౌంట్లు ఓపెన్ చేద్దాం. వేరియబుల్ పేర్లు కూడా వాళ్ళ పేర్లే పెడదాం (`a1`, `a2` అని కాకుండా).


```java
// అకౌంట్ 1 (పంపేవాడు - Sender)
Account raju = new Account();
raju.name = "Raju";
raju.balance = 1000;


// అకౌంట్ 2 (తీసుకునేవాడు - Receiver)
Account ravi = new Account();
ravi.name = "Ravi";
ravi.balance = 300;
```


**మన లక్ష్యం:**
రాజు, రవికి **₹500** పంపాలి.


**లాజిక్:**
దీనికోసం మనం `transfer()` అనే మెథడ్ రాయాలి. అందులో రెండు పనులు జరగాలి:


1. రాజు బ్యాలెన్స్ నుంచి 500 తగ్గాలి.
2. రవి బ్యాలెన్స్ కి 500 కలవాలి.


-----


##  2️⃣ ట్రాన్స్‌ఫర్ జరగక ముందు పరిస్థితి 


 * **Object `raju`:** రాజు డేటా (1000) కలిగి ఉంది.
 * **Object `ravi`:** రవి డేటా (300) కలిగి ఉంది.
 * **ముఖ్య గమనిక:** ఇవి రెండూ మెమరీలో వేరు వేరు బాక్సులు. ఒకదానితో ఒకటికి సంబంధం లేదు.


-----


##  3️⃣ కోడ్ ఎలా రాయాలి? 


మనం ఆబ్జెక్ట్ పేర్లు మనుషుల పేర్లతోనే పెట్టాం కాబట్టి, కోడ్ చదువుతుంటే ఒక ఇంగ్లీష్ వాక్యంలా ఉంటుంది:


```java
raju.transfer(ravi, 500);
```


**దీని అర్థం:**
**"ఓ రాజు (`raju`), నువ్వు రవికి (`ravi`) 500 ట్రాన్స్‌ఫర్ చేయి."**


ఇక్కడ పాత్రలు చాలా క్లియర్ గా ఉన్నాయి:
✔ **`raju`** = Sender (మెథడ్ ని పిలిచినవాడు / పంపేవాడు).
✔ **`ravi`** = Receiver (బ్రాకెట్ లోపల ఉన్నవాడు / తీసుకునేవాడు).


**విద్యార్థులకు వచ్చే అసలైన సందేహం:**
*"సార్, మెథడ్ లోపల 'raju' అనే పేరు కనిపించదు కదా? కేవలం `balance` అని మాత్రమే ఉంటుంది. మరి పంపేది రాజే అని జావాకి ఎలా తెలుస్తుంది?"*


జావాకి అర్థం కావడానికి ఒక రూల్ ఉంది:


> **"ఎవరైతే మెథడ్ ని పిలుస్తారో (Call), వాళ్ళే Sender."**


ఈ Sender ని గుర్తించడానికే మనం **`this`** వాడతాం.


-----


## 4️⃣ `this` అంటే ఏంటి? (సింపుల్ డెఫినిషన్)


> **`this` అంటే, ప్రస్తుతం ఏ ఆబ్జెక్ట్ అయితే మెథడ్ ని రన్ చేస్తుందో, ఆ ఆబ్జెక్ట్.**


అంటే:


 * ఒకవేళ `raju.transfer(...)` అని పిలిస్తే → మెథడ్ లోపల **`this` అంటే `raju`**.
 * ఒకవేళ `ravi.transfer(...)` అని పిలిస్తే → మెథడ్ లోపల **`this` అంటే `ravi`**.


-----


## 5️⃣ మెథడ్ లోపల ఏం జరుగుతుంది?


ట్రాన్స్‌ఫర్ అంటే రెండు స్టెప్స్:


1. **Sender (`this`)** డబ్బులు పోగొట్టుకుంటాడు.
2. **Receiver (`parameter`)** డబ్బులు పొందుతాడు.


<!-- end list -->


```java
class Account {
   String name;
   int balance;


   void transfer(Account receiver, int amount) {
       this.balance = this.balance - amount;         // Step 1: పంపేవాడు (this) ఇస్తున్నాడు
       receiver.balance = receiver.balance + amount; // Step 2: తీసుకునేవాడికి (receiver) వస్తోంది
   }
}
```


-----


## 6️⃣ లైన్-బై-లైన్ వివరణ (పేర్లతో సహా)


`void transfer(Account receiver, int amount)`


 * **`receiver`**: బ్రాకెట్ లోపల మనం ఎవరిని పంపాం? **`ravi`**.
 * **`amount`**: 500.


`this.balance = this.balance - amount;`


 * **`this`**: మెథడ్ ని ఎవరు పిలిచారు? **`raju`**.
 * **లాజిక్:** "రాజు బ్యాలెన్స్ లోంచి 500 తీసేయి."


`receiver.balance = receiver.balance + amount;`


 * **`receiver`**: రిసీవర్ ఎవరు? **`ravi`**.
 * **లాజిక్:** "రవి బ్యాలెన్స్ కి 500 కలుపు."


-----


## 7️⃣ ఫ్లో చార్ట్ 


**`raju`** మెథడ్ ని కాల్ చేసాడు ->జావా మెథడ్ లోపలికి వెళ్ళింది. జావా అనుకుంటుంది: **"ఓకే, పిలిచింది రాజు కాబట్టి, `this` అంటే `raju`."** -> లైన్ 1: `raju` బ్యాలెన్స్ తగ్గింది. -> లైన్ 2: `ravi` బ్యాలెన్స్ పెరిగింది.    -> పని పూర్తయింది\!


-----


## 8️⃣ పూర్తి ప్రోగ్రామ్ (క్లాస్ రూమ్ కోసం)


ఇదిగోండి ఫైనల్ కోడ్. `main` method ఇప్పుడు ఎంత క్లియర్ గా ఉందో చూడండి\!


```java
class Account {
   String name;
   int balance;


   // డబ్బులు పంపే మెథడ్
   void transfer(Account receiver, int amount) {
       this.balance = this.balance - amount;
       receiver.balance = receiver.balance + amount;
       System.out.println("✅ Transfer Successful!");
   }
}


public class BankApp {
   public static void main(String[] args) {


       // 1. రాజుని సృష్టిద్దాం
       Account raju = new Account();
       raju.name = "Raju";
       raju.balance = 1000;


       // 2. రవిని సృష్టిద్దాం
       Account ravi = new Account();
       ravi.name = "Ravi";
       ravi.balance = 300;


       // 3. ట్రాన్స్‌ఫర్ 
       System.out.println("--- Initiating Transfer ---");
      
       // "రాజు, రవికి 500 పంపుతున్నాడు"
       raju.transfer(ravi, 500);  


       // 4. ఫైనల్ బ్యాలెన్స్ చెక్ చేద్దాం
       System.out.println(raju.name + " Balance: " + raju.balance); // 500 ఉండాలి
       System.out.println(ravi.name + " Balance: " + ravi.balance); // 800 అవ్వాలి
   }
}
```


-----


##  9️⃣ ముగింపు


 * **వేరియబుల్ పేర్లు:** `a1`, `a2` అని కాకుండా `raju`, `ravi` అని పేర్లు పెట్టడం వల్ల కోడ్ అర్థం చేసుకోవడం చాలా ఈజీ అయ్యింది.
 * **బంగారు సూత్రం (Golden Rule):** మెథడ్ లోపల ఉన్న కోడ్, ఆ మెథడ్ ని పిలిచిన ఆబ్జెక్ట్ కి సొంతం.
 * **కీవర్డ్:** ఆ సొంత ఆబ్జెక్ట్ ని చూపించడానికే `this` వాడతాం.
 * **ఈ ఉదాహరణలో:**
     * **`this`** → `raju` (పంపేవాడు)
     * **`receiver`** → `ravi` (తీసుకునేవాడు)
