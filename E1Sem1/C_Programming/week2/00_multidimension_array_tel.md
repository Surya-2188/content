

## 1\. 2D Arrays ఎందుకు కావాలి? (సమస్య ఏంటి?)


మనం ఇంతకుముందు **1D Arrays** గురించి చూసాం. స్టూడెంట్స్ మార్కులు లేదా లిస్ట్ స్టోర్ చేయడానికి అవి సూపర్.
కానీ, ఒకవేళ మీరు ఒక **అటెండెన్స్ రిజిస్టర్ (Attendance Register)** మెయింటైన్ చేయాలనుకోండి.


అందులో:


 * 60 మంది స్టూడెంట్స్ ఉంటారు (Rows).
 * 30 రోజులు ఉంటాయి (Columns).


దీన్ని మామూలు అరేలో స్టోర్ చేయలేం. ఎందుకంటే అది ఒక లైన్ మాత్రమే. మనకు కావాల్సింది ఒక **టేబుల్ (Table)** లాంటి స్ట్రక్చర్.


## 2\. పరిష్కారం: Multidimensional Arrays


టేబుల్ లాంటి డేటాని స్టోర్ చేయడానికి సి లాంగ్వేజ్ లో **2D Arrays** వాడతాం.


 * **1D Array:** ఒక లైన్ లాంటిది.
 * **2D Array:** ఒక బాక్స్ లేదా టేబుల్ లాంటిది (Rows & Columns).


## 3\. సింటాక్స్ (Syntax)


రెండు సైజులు చెప్పాలి: ఎన్ని అడ్డ గీతలు (Rows), ఎన్ని నిలువు గీతలు (Columns).


```c
int attendance[60][30];
```


 * **60:** స్టూడెంట్స్ కోసం (Rows).
 * **30:** రోజుల కోసం (Columns).


యాక్సెస్ చేయడం ఇలా: `attendance[0][0]` (మొదటి స్టూడెంట్, మొదటి రోజు).


## 4\. మెమరీలో ఎలా ఉంటుంది? (Memory View)


మనం దీన్ని ఒక బాక్స్ లా ఊహించుకుంటాం కానీ, కంప్యూటర్ లోపల మెమరీ ఎప్పుడూ ఒకే లైన్ లో ఉంటుంది.


దీన్నే **Row-Major Order** అంటారు.


1. ముందు **Row 0** మొత్తం వరుసగా స్టోర్ అవుతుంది.
2. దాని పక్కనే **Row 1** మొత్తం స్టోర్ అవుతుంది.
3. ఆ తర్వాత **Row 2**... ఇలా సాగుతుంది.


## 5\. లూప్స్ ఎలా వాడాలి? (Nested Loops)


టేబుల్ లో డేటా నింపాలంటే మనకు **రెండు లూప్స్** కావాలి.


1. **బయటి లూప్ (Outer Loop):** Rows కోసం (స్టూడెంట్స్ ని మార్చడానికి).
2. **లోపలి లూప్ (Inner Loop):** Columns కోసం (రోజుల్ని మార్చడానికి).


<!-- end list -->


```c
for(i = 0; i < 2; i++) {       // Rows
   for(j = 0; j < 2; j++) {   // Columns
       scanf("%d", &matrix[i][j]);
   }
}
```


ఇలా లూప్ లోపల లూప్ వాడితేనే మనం మ్యాట్రిక్స్ ని హ్యాండిల్ చేయగలం.


## 6\. సమ్మరీ (Summary)


 * డేటా టేబుల్ లాగా ఉంటే **2D Array** వాడాలి.
 * దీనికి రెండు ఇండెక్స్ లు ఉంటాయి `arr[row][col]`.
 * మెమరీలో లైన్ గానే స్టోర్ అయినా, మనం టేబుల్ లా ఫీల్ అయ్యి వాడుకుంటాం.
 * ఇన్పుట్/అవుట్పుట్ కోసం **రెండు లూప్స్** కచ్చితంగా కావాలి.
