

#  అధ్యాయం: C లో ఫంక్షన్స్ (Functions) పరిచయం


## 1\. మనకు C లో Functions ఎందుకు అవసరం?


మనం ల్యాబ్‌లో చేసే ఒక సాధారణ సమస్యతో దీన్ని మొదలుపెడదాం.


### 1.1 `main` లో ఒకే కోడ్‌ని మళ్ళీ మళ్ళీ రాయడం (Repetition)


మనం చేయాల్సిన పని:


1. ముగ్గురు స్టూడెంట్స్ యొక్క మార్కులను తీసుకోవాలి.
2. ప్రతి స్టూడెంట్ 3 సబ్జెక్టుల మార్కులను ఎంటర్ చేస్తారు.
3. ప్రతి స్టూడెంట్ కి టోటల్ (Total), యావరేజ్ (Average) కనుగొని, వారు PASS అయ్యారా లేదా FAIL అయ్యారా అని చెప్పాలి.


Functions వాడకుండా సాధారణ C ప్రోగ్రామ్ రాస్తే అది ఇలా ఉంటుంది:


```c
#include <stdio.h>


int main(void) {
   int m1, m2, m3;   // 3 సబ్జెక్టుల మార్కులు
   int total;        // టోటల్ మార్కులు
   float avg;        // యావరేజ్ మార్కులు


   // ----- స్టూడెంట్ 1 -----
   printf("Enter marks of student 1 (3 subjects): ");
   scanf("%d%d%d", &m1, &m2, &m3);
   total = m1 + m2 + m3;
   avg = total / 3.0f;
   printf("Student 1: Average = %.2f -> %s\n",
          avg,
          (avg >= 40.0f) ? "PASS" : "FAIL");


   // ----- స్టూడెంట్ 2 -----
   printf("Enter marks of student 2 (3 subjects): ");
   scanf("%d%d%d", &m1, &m2, &m3);
   total = m1 + m2 + m3;
   avg = total / 3.0f;
   printf("Student 2: Average = %.2f -> %s\n",
          avg,
          (avg >= 40.0f) ? "PASS" : "FAIL");


   // ----- స్టూడెంట్ 3 -----
   printf("Enter marks of student 3 (3 subjects): ");
   scanf("%d%d%d", &m1, &m2, &m3);
   total = m1 + m2 + m3;
   avg = total / 3.0f;
   printf("Student 3: Average = %.2f -> %s\n",
          avg,
          (avg >= 40.0f) ? "PASS" : "FAIL");


   return 0;
}
```


ఇక్కడ `main` లోపల ప్రతి స్టూడెంట్ కోసం:


1. మనం 3 మార్కులు తీసుకుంటున్నాం (`scanf`).
2. Total కనుగొంటున్నాం.
3. Average కనుగొంటున్నాం.
4. PASS/FAIL అని ప్రింట్ చేస్తున్నాం.


లాజిక్ (Logic) అనేది స్టూడెంట్ 1, 2 మరియు 3 కి **ఒకేలా ఉంది**. కేవలం మార్కులు మరియు స్టూడెంట్ నంబర్ మాత్రమే మారుతున్నాయి.


**అసలు సమస్య:**
ఒకవేళ రేపు మనం **50 మంది స్టూడెంట్స్** కి ఇదే పని చేయాలనుకుంటే? లేదా ఇదే లాజిక్ ప్రోగ్రామ్‌లో మరో చోట వాడాలనుకుంటే?


 * కోడ్ చాలా పెద్దగా, గందరగోళంగా (messy) మారుతుంది.
 * ఒకవేళ మనం రూల్ మార్చాలనుకుంటే (ఉదాహరణకు PASS మార్కులు 35 కి తగ్గితే), మనం అన్ని చోట్లా మార్చాల్సి ఉంటుంది.
 * తప్పులు (Errors) సరిదిద్దడం కష్టమవుతుంది.


కాబట్టి మనకు ఒక పరిష్కారం కావాలి: **ఒక నిర్దిష్టమైన రూల్ (Rule) ని ఒక్కసారే రాసి, దాన్ని అనేక రకాల ఇన్‌పుట్స్ (Inputs) కి వాడగలగాలి.** దీని కోసమే మనకు **Functions** అవసరం.

## 2\. మ్యాథమెటిక్స్ ఉదాహరణ: Functions అనేవి రూల్స్ లాంటివి


C లో ఫంక్షన్స్ గురించి మాట్లాడే ముందు, మ్యాథ్స్ (Maths) లో మనం ఏం చేస్తామో గుర్తుచేసుకుందాం.


### 2.1 మ్యాథ్స్‌లో ఫంక్షన్స్


మ్యాథ్స్‌లో, ఎప్పుడైతే $x$ మరియు $y$ మధ్య ఒకే రకమైన సంబంధం (relation) ఉంటుందో, అప్పుడు మనం ఒక Function ని డిఫైన్ చేస్తాం.


**ఉదాహరణ:**
$$y = 2x + 3$$


మనం ప్రతిసారీ "$(2x + 3)$" అని రాయం. దానికి బదులుగా ఇలా డిఫైన్ చేస్తాం:
$$f(x) = 2x + 3$$


ఇప్పుడు:


 * $f(3)$ అంటే: $x = 3$ అయినప్పుడు ఆ రూల్ ని అప్లై చేయడం.
 * $f(10)$ అంటే: $x = 10$ అయినప్పుడు ఆ రూల్ ని అప్లై చేయడం.


**ఒకే రూల్, అనేక ఇన్‌పుట్స్ $\rightarrow$ అనేక అవుట్‌పుట్స్.**
ప్రోగ్రామింగ్ ఫంక్షన్స్ కూడా ఇదే ఐడియాతో పనిచేస్తాయి.


### 2.2 ఉదాహరణ 1: $\sin^2 x + \cos^2 x = 1$ ని చెక్ చేయడం


దీన్ని C ప్రోగ్రామ్‌లో రాద్దాం.


#### 2.2.1 $f(x) = \sin^2 x + \cos^2 x$ కోసం C ఫంక్షన్


```c
#include <stdio.h>
#include <math.h>


// Function definition: f(x) = sin^2(x) + cos^2(x)
double trig_identity(double x) {
   double s = sin(x);           // లోకల్ వేరియబుల్: x యొక్క sine
   double c = cos(x);           // లోకల్ వేరియబుల్: x యొక్క cosine
   double value = s * s + c * c;
   return value;                // లెక్కించిన విలువను వెనక్కి పంపడం
}


int main(void) {
   double x;


   x = 0.0;
   printf("x = 0.0,   f(x) = %.6f\n", trig_identity(x));


   x = M_PI / 6; // π/6
   printf("x = pi/6,  f(x) = %.6f\n", trig_identity(x));


   return 0;
}
```


#### 2.2.2 లైన్ బై లైన్ వివరణ


 * `double trig_identity(double x)`
     * `double` $\rightarrow$ **Return Type**: ఈ ఫంక్షన్ చివరికి ఒక `double` విలువను ఇస్తుంది.
     * `trig_identity` $\rightarrow$ **Function Name** (ఫంక్షన్ పేరు).
     * `(double x)` $\rightarrow$ **Parameter List**:
         * `x` అనేది పారామీటర్ (ఇన్‌పుట్). దీని టైప్ `double`.
 * `double s = sin(x);`
     * `s` మరియు `c` అనేవి ఫంక్షన్ లోపల ఉండే **Local Variables**.
 * `return value;`
     * చివరి ఫలితాన్ని `main` కి పంపడం.


`main` లో మనం `trig_identity(x)` అని రాసినప్పుడు, అది మ్యాథ్స్‌లో $f(x)$ లాగే పనిచేస్తుంది.

## 3\. Function Definition మరియు Function Call


ఇప్పుడు C భాషలో వాడే పదాలను చూద్దాం.


### 3.1 Function Definition (నిర్వచనం)


దీని నిర్మాణం ఇలా ఉంటుంది:


```c
return_type function_name(parameter_type1 param1, ...) {
   // లోకల్ వేరియబుల్స్
   // బాడీ (Instructions)
   return expression;
}
```


కంప్యూటర్ (Compiler) వీటిని చెక్ చేస్తుంది:


1. **Return type:** (ఉదా. `int`, `double`, `void`). ఫంక్షన్ ఏ రకమైన విలువను తిరిగి ఇస్తుంది?
2. **Function Name:** ఫంక్షన్ పేరు.
3. **Parameter List:** బ్రాకెట్ల `(...)` లోపల ఉండేవి. ఎన్ని ఇన్పుట్స్ కావాలి? వాటి టైప్ ఏంటి?
4. **Function Body:** `{ ... }` లోపల ఉండే కోడ్.


### 3.2 Function Call (పిలవడం)


మనం `main` లో ఫంక్షన్‌ని వాడటాన్ని "Call" అంటాం.


```c
result = function_name(arg1, arg2, ...);
```


ఫంక్షన్ కాల్ దగ్గర కంపైలర్ ఏం చెక్ చేస్తుంది?


1. ఆ పేరుతో ఫంక్షన్ ఉందా?
2. దాని రిటర్న్ టైప్ ఏంటి?
3. ఎన్ని పారామీటర్స్ ఉన్నాయి? మనం ఇచ్చిన విలువలు (Arguments) వాటికి సరిపోతున్నాయా?


-----


## 4\. Parameters (పారామీటర్స్) vs Arguments (ఆర్గ్యుమెంట్స్)


ఇది చాలా ముఖ్యం:


 * **Function Definition లో:**
   `double trig_identity(double x)`
   ఇక్కడ `x` అనేది **Parameter**. ఇది మ్యాథ్స్‌లో $f(x)$ లోని $x$ లాంటిది. ఇది కేవలం ఒక పేరు (Placeholder).


 * **Function Call లో:**
   `value = trig_identity(M_PI / 4);`
   ఇక్కడ `M_PI / 4` అనేది **Argument**. ఇది మనం పంపిస్తున్న **నిజమైన విలువ (Actual Value)**.


> **గుర్తుంచుకోండి:**
>
>   * **Parameters:** డెఫినిషన్‌లో ఉండే వేరియబుల్ పేర్లు.
>   * **Arguments:** కాల్ చేసేటప్పుడు మనం పంపించే విలువలు.


-----


## 5\. Parameters గా ఏ డేటా టైప్స్ వాడవచ్చు?


C లో ప్రాథమిక డేటా టైప్స్ అన్నింటినీ పారామీటర్స్ గా వాడవచ్చు:


 * `int`: సంఖ్యలు, కౌంట్స్ కోసం.
 * `double` / `float`: పాయింట్ విలువలు (marks, probabilities) కోసం.
 * `char`: అక్షరాలు (grades) కోసం.


-----


## 6\. Scope: వేరియబుల్ ఎక్కడ పనిచేస్తుంది?


C ప్రోగ్రామ్ అంటే **Variables + Instructions**.
కానీ, **ప్రతి వేరియబుల్ కి ఒక Scope (పరిధి) ఉంటుంది.**


### 6.1 Scope అంటే ఏమిటి?


**Scope of a variable = ప్రోగ్రామ్‌లో ఆ వేరియబుల్‌ని మనం ఎక్కడ వాడగలము (Access/Update) అనేది తెలిపేది.**


ఒక వేరియబుల్ యొక్క స్కోప్ లోపల మాత్రమే మనం:


1. దాని విలువను చూడగలం (Access).
2. దాని విలువను మార్చగలం (Update).


### 6.2 Local Variables (లోకల్ వేరియబుల్స్)


ఇవి ఒక ఫంక్షన్ లేదా బ్లాక్ `{ ... }` లోపల డిక్లేర్ చేయబడతాయి.


 * **Scope:** ఆ బ్లాక్ మొదలైన దగ్గర నుండి `{` ఆ బ్లాక్ ముగిసే `}` వరకు మాత్రమే.
 * **ముఖ్య గమనిక:** వేరే ఫంక్షన్‌లో ఉన్న లోకల్ వేరియబుల్స్ ఇక్కడ పనిచేయవు.

```c
double square(double x) {
   double result = x * x;      // ఇది square కి మాత్రమే లోకల్
   return result;
}


int main(void) {
   double y = 3.0;
   // printf("%f", result);    // ❌ Error! 'result' అనేది main లో లేదు.
   return 0;
}
```


### 6.3 Global Variables (గ్లోబల్ వేరియబుల్స్)


ఇవి అన్ని ఫంక్షన్స్ బయట డిక్లేర్ చేయబడతాయి. ఇవి ఫైల్ మొత్తం ఎక్కడైనా పనిచేస్తాయి.


### 6.4 Parameters కూడా Local Variables లాంటివే


`double trig_identity(double x)` లో, `x` అనేది పారామీటర్ అయినా కూడా, అది ఆ ఫంక్షన్‌కి **Local Variable**. దాన్ని ఆ ఫంక్షన్ బయట వాడలేము.

## 7\. Library Functions vs మన సొంత Functions


ఇప్పటి వరకు మనం `trig_identity`, `mean3` వంటి మన సొంత ఫంక్షన్స్ రాసాం (User-defined functions).
కానీ, `printf`, `scanf`, `sin`, `cos` అనేవి కూడా ఫంక్షలే.


వీటిని **Built-in Library Functions** అంటారు.


 * ఇవి C లైబ్రరీలో ముందే రాయబడి ఉన్నాయి.
 * మనం కేవలం వాటిని **Call** చేస్తున్నాం.


**మనం `#include <stdio.h>` ఎందుకు రాస్తాం?**
`printf` అంటే ఏంటి? దానికి ఎన్ని ఇన్పుట్స్ కావాలి? అనే **Definition**, `stdio.h` అనే హెడర్ ఫైల్‌లో ఉంటుంది. అది లేకపోతే కంపైలర్ కి `printf` గురించి తెలియదు, అందుకే ఎర్రర్ (Error) ఇస్తుంది.

## 8\. విద్యార్థులు చేసే సాధారణ తప్పులు (Common Mistakes)


వీటిని చూసి సరిదిద్దుకోండి:


1. **Too few arguments:** ఫంక్షన్ కి 2 విలువలు కావాలంటే 1 మాత్రమే ఇవ్వడం.
   ```c
   sum(10); // ❌ Error
   ```
2. **Wrong type:** `int` అడిగిన చోట `double` (3.5) ఇవ్వడం.
3. **Void function return:** ఏమీ రిటర్న్ చేయని (`void`) ఫంక్షన్ నుండి విలువను ఆశించడం.
   ```c
   y = show_double(5); // ❌ Error
   ```
4. **Conflicting declarations:** పైన డిక్లేర్ చేసిన టైప్ ఒకటి, కింద రాసిన డెఫినిషన్ టైప్ మరొకటి ఉండటం.
5. **Semicolon Mistake:** ఫంక్షన్ డెఫినిషన్ హెడర్ చివర సెమీకోలన్ పెట్టడం.
   ```c
   int square(int x); // ❌ ఇక్కడ ; ఉండకూడదు (డెఫినిషన్ రాసేటప్పుడు)
   {
       return x*x;
   }
   ```
6. **Uninitialized variable:** విలువ లేని (Garbage value) వేరియబుల్ ని ఫంక్షన్ కి పంపడం.


-----


## 9\. సారాంశం (Summary)


 * C ప్రోగ్రామ్ అనేది **Variables** మరియు **Instructions** కలయిక.
 * ప్రతి వేరియబుల్ కి ఒక **Scope** ఉంటుంది. అది ఎక్కడ పనిస్తుందో అదే నిర్ణయిస్తుంది.
 * ఒకే పనిని మళ్ళీ మళ్ళీ చేయాల్సి వచ్చినప్పుడు, **Function** ని వాడతాం.
 * **Function Definition:** రూల్ ని తయారు చేయడం (Return type, Name, Parameters).
 * **Function Call:** ఆ రూల్ ని వాడటం (Arguments పంపడం).
 * `printf` వంటివి **Built-in Functions**. వాటి వివరాలు హెడర్ ఫైల్స్ (`stdio.h`) లో ఉంటాయి.
